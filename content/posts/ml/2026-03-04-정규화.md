머신러닝 분야를 공부하다 보면 **정규화**라는 용어를 매우 자주 접하게 된다. 그러나 많은 IT도서에서 Normalization, Standardization, Regularization을 모두 정규화라고 번역하여 독자들로 하여금 이해를 하기 어렵게 한다. 최근 도서는 이를 구분하여 옆에 병기를 해놓기도 하지만 항상 헷갈리는 주제이기에 이를 정리해서 남겨두고자 한다.

## 한눈에 비교하기

| 항목      | Normalization         | Standardization           | Regularization |
| --------- | --------------------- | ------------------------- | -------------- |
| 목적      | 값 범위를 0~1​로 압축 | 평균 0, 표준편차 1로 변환 | 과적합 방지    |
| 적용 시점 | 데이터 전처리         | 데이터 전처리             | 모델 학습 중   |
| 대상      | 입력 데이터 피처      | 입력 데이터 피처          | 모델 가중치    |
| 분포 변화 | 분포 유지             | 정규분포로 변환           | 데이터 무관    |

---

## Normalization

$$X_{norm}= \frac {X-min(X)}{max(X)-min(X)}$$

```python
from sklearn.preprocessing import MinMaxScaler
scaler = MinMaxScaler()
X_scaled = scaler.fit_transform(X)
```

- 데이터 값의 범위를 **0~1**의 값으로 압축
- 분포가 정규분포가 아닐때 좋다
- 추천 알고리즘: **k-NN, 신경망(이미지 픽셀), 거리 기반 알고리즘**
> [!WARNING]
> **이상치에 민감**  
> 극단값이 있으면 나머지 값들이 좁은 범위에 몰릴 수 있음

---

## Standardization

$$Z=\frac{x-\mu}{\sigma}$$

```python
from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)
```

- 값에 평균을 빼고 표준편차로 나눠 **Z-score**로 변환
- 데이터가 정규분포에 가까울 때 가장 효과적
- 이상치 영향이 상대적으로 적음
- 추천 알고리즘: **SVM, PCA, 선형 회귀, 로지스틱 회귀**

> [!TIP]
> Standardization으로 이상치 제거 → Normalization으로 스케일 조정 순서로 진행하는 경우가 많다

---

## Regularization

테이터가 아닌 **모델의 손실 함수**에 패널티 항을 추가해 가중치가 과도하게 커지는 것을 억제

|           | L1 (Lasso)                                                      | L2 (Ridge)                                                  |
| --------- | --------------------------------------------------------------- | ----------------------------------------------------------- |
| 모델 형태 | Sparse (희소)                                                   | Dense (밀집)                                                |
| 페널티 항 | $\lambda \sum \lVert w_i \rVert$                            | $\lambda \sum {w_i}^2$                                    |
| 효과      | 일부 가중치를 0으로 만들어 피처 선택                            | 가중치를 작게 만들어 부드러운 모델                          |
| 사용 시   | 피처가 많고(수백~수만), 대부분 무관계                           | 대부분의 피처가 유효할 때                                   |
| 예시      | 텍스트 분류 (단어 10만 개 중 수백 개만 유의미)                  | 집값 예측 (면적, 층수, 위치, 연식 등 전부 관련)             |

---

## 언제 뭘 쓸까

> [!TIP]
> **Normalization** -> 데이터 스케일 차이가 크고 분포가 비정규  
> **Standardization** -> 데이터가 정규분포에 가깝거나 PCA/SVM 사용 시  
> **Regularization** -> 모델이 훈련 데이터에 과적합
